## Overview

Direct Logit Attribution (DLA) decomposes the final logit difference into additive contributions from each model component (attention heads, MLP layers, embeddings). Unlike activation patching which measures **causal importance**, DLA measures each component's **direct linear contribution** to the output.

### The Compute → Readout Dissociation

A key finding from DLA analysis is that **computation heads** and **readout heads** often differ:

- **Compute heads**: High patching importance but low DLA — they process information but don't directly write to output
- **Readout heads**: High DLA but moderate patching — they read processed info and write the decision

<Note>
Both types are necessary for the circuit, but only readout heads show up strongly in DLA scores.
</Note>

## How It Works

For each component **c**, its DLA score is:

```
DLA(c) = (output_c @ W_U[:, tool_token]) - (output_c @ W_U[:, no_tool_token])
```

where `W_U` is the unembedding matrix.

### Why This Works

The residual stream in a transformer is a linear sum of component outputs:

```
residual = embed + pos_embed + Σ(head_outputs) + Σ(mlp_outputs)
```

Since the final logits are a linear function of the residual stream:

```
logits = residual @ W_U + bias
```

The decomposition is **exact** — the sum of all component DLA scores equals the total logit difference (up to bias/LayerNorm terms).

## Implementation

### Computing the Logit Direction

```python logit_attribution.py:105-119
def _get_logit_direction(
    model: HookedTransformer,
    tool_token_id: int,
    no_tool_token_id: int,
) -> Float[Tensor, "d_model"]:
    """Compute the 'logit difference direction' in residual stream space.
    
    This is W_U[:, tool_token] - W_U[:, no_tool_token], the direction in d_model space
    that, when you dot-product with a component's output, gives that component's
    contribution to the logit difference.
    """
    W_U = model.W_U  # shape [d_model, vocab_size]
    return W_U[:, tool_token_id] - W_U[:, no_tool_token_id]
```

### Extracting Head Outputs

TransformerLens caches pre-output-projection activations, so we manually apply the output projection:

```python logit_attribution.py:74-98
def _get_head_output(
    model: HookedTransformer,
    cache: dict,
    layer: int,
    head: int,
    position: int,
) -> Float[Tensor, "d_model"]:
    """Extract a single attention head's output at a given position, in d_model space.
    
    TransformerLens caches hook_z (pre-W_O, shape [batch, pos, head, d_head]) but not
    the per-head post-W_O result. We manually project through W_O[layer, head] to get
    the head's contribution in residual stream (d_model) space for DLA decomposition.
    """
    key = f"blocks.{layer}.attn.hook_z"
    # Shape: [batch, pos, n_heads, d_head]
    z = cache[key][:, position, head, :]  # [batch, d_head]

    # W_O[layer, head] has shape [d_head, d_model]
    W_O_head = model.W_O[layer, head]
    head_output = z @ W_O_head  # [batch, d_model]

    return head_output.mean(dim=0)
```

### Full Decomposition

```python logit_attribution.py:126-208
def compute_dla(
    model: HookedTransformer,
    prompts: list[str],
    tokenizer: PreTrainedTokenizerFast,
    config: ExperimentConfig = CONFIG,
) -> dict:
    """Decompose the logit difference into per-component direct contributions.
    
    Returns dict with:
        - "head_dla": np.ndarray (n_layers, n_heads) — per-head DLA scores
        - "mlp_dla": np.ndarray (n_layers,) — per-MLP-layer DLA scores
        - "embed_dla": float — token + positional embedding contribution
    """
    if not prompts:
        raise ValueError("prompts must be non-empty")

    n_layers = config.model.n_layers
    n_heads = config.model.n_heads
    tool_token_id = tokenizer.convert_tokens_to_ids(config.model.tool_call_token)
    no_tool_token_id = tokenizer.convert_tokens_to_ids(config.model.no_tool_token)

    logit_dir = _get_logit_direction(model, tool_token_id, no_tool_token_id)

    # Accumulators
    head_dla = np.zeros((n_layers, n_heads), dtype=np.float64)
    mlp_dla = np.zeros(n_layers, dtype=np.float64)
    embed_dla = 0.0

    device = next(model.parameters()).device

    for prompt in prompts:
        tokens = tokenizer(prompt, return_tensors="pt").to(device)
        _, cache = model.run_with_cache(tokens["input_ids"])

        # Decision position: last token
        seq_len = tokens["input_ids"].shape[1]
        pos = seq_len - 1

        # Per-head contributions
        for layer in range(n_layers):
            for head in range(n_heads):
                head_out = _get_head_output(model, cache, layer, head, pos)
                head_dla[layer, head] += head_out @ logit_dir

            # MLP contribution
            mlp_out = _get_component_output(cache, f"blocks.{layer}.hook_mlp_out", pos)
            mlp_dla[layer] += mlp_out @ logit_dir

        # Embedding contribution (token + positional)
        token_embed = _get_component_output(cache, "hook_embed", pos)
        pos_embed = _get_component_output(cache, "hook_pos_embed", pos)
        embed_dla += ((token_embed + pos_embed) @ logit_dir).item()

        clear_gpu_cache()

    # Average across prompts
    n = len(prompts)
    head_dla /= n
    mlp_dla /= n
    embed_dla /= n

    return {
        "head_dla": head_dla,
        "mlp_dla": mlp_dla,
        "embed_dla": embed_dla,
    }
```

## Sanity Check: Reconstruction Error

Since the decomposition is exact, we can verify correctness:

```python logit_attribution.py:215-264
def decompose_single_example(
    model: HookedTransformer,
    prompt: str,
    tokenizer: PreTrainedTokenizerFast,
    config: ExperimentConfig = CONFIG,
) -> dict:
    """Full DLA decomposition for a single example with reconstruction error check."""
    result = compute_dla(model, [prompt], tokenizer, config)

    # Compute actual total logit diff
    device = next(model.parameters()).device
    tokens = tokenizer(prompt, return_tensors="pt").to(device)
    logits = model(tokens["input_ids"])

    tool_token_id = tokenizer.convert_tokens_to_ids(config.model.tool_call_token)
    no_tool_token_id = tokenizer.convert_tokens_to_ids(config.model.no_tool_token)

    last_logits = logits[:, -1, :]
    total_diff = (last_logits[:, tool_token_id] - last_logits[:, no_tool_token_id]).mean().item()

    component_sum = (
        result["head_dla"].sum()
        + result["mlp_dla"].sum()
        + result["embed_dla"]
    )

    result["total_logit_diff"] = total_diff
    result["reconstruction_error"] = abs(total_diff - component_sum)

    return result
```

<Check>
**Expected result**: Reconstruction error should be < 0.01, confirming the decomposition is mathematically exact.
</Check>

## Example Results

For a typical tool-calling example:

| Component | DLA Score | Interpretation |
|-----------|-----------|----------------|
| Layer 11, Head 3 | +1.82 | Strong direct contribution toward tool-call |
| Layer 10, Head 7 | +0.45 | Moderate contribution (but high patching score!) |
| Layer 6, MLP | +0.92 | MLP processes numerical info |
| Embeddings | -0.33 | Initial bias against tool-calling |

<Info>
Notice Layer 10 Head 7 has moderate DLA but could have high patching importance — this is a **compute head** that enables other heads rather than directly writing the answer.
</Info>

## Use Cases

1. **Circuit Discovery**: Combine with patching scores to identify both compute and readout heads
2. **Feature Attribution**: Identify which heads write specific features to the residual stream
3. **Debugging**: Verify that interpretability claims are mechanistically grounded
