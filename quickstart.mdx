# Quickstart Guide

Get up and running with the tool-calling mechanistic interpretability pipeline in under 10 minutes.

## Prerequisites

- Python 3.8+
- CUDA-capable GPU recommended (RTX 3060 12GB or equivalent)
- CPU-only execution supported (slower)

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <your-repo-url>
    cd tool-call-mech-interp
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs:
    - PyTorch ≥2.1.0
    - TransformerLens ≥1.14.0 (for mechanistic interpretability)
    - SAE Lens ≥3.0.0 (for sparse autoencoder analysis)
    - Transformers ≥4.35.0 (for GPT-2 fine-tuning)
    - Visualization tools (plotly, kaleido)
  </Step>

  <Step title="Verify installation">
    ```bash
    python -c "import torch; import transformer_lens; print('Setup OK')"
    ```
  </Step>
</Steps>

## Run the Full Pipeline

The pipeline runs all phases sequentially: data generation, fine-tuning, interpretability analysis, and visualization.

```bash
python run_pipeline.py
```

<Info>
**Runtime**: ~5 minutes on RTX 3060 12GB, ~15-20 minutes on CPU
</Info>

### What This Does

<Steps>
  <Step title="Phase 1-2: Data Generation">
    Generates 10,000 adversarially-balanced examples:
    - 5,000 tool-call prompts (requiring calculator/lookup/datetime/unit_conversion)
    - 5,000 no-tool prompts (conceptual questions, definitions)
    - Balanced for confounds: numbers present 59.9% vs 59.4%, temporal keywords 34.3% vs 29.4%
    - Saves to `results/data/dataset.json`
  </Step>

  <Step title="Phase 3: Fine-Tuning">
    Trains GPT-2 Small to predict `<|tool_call|>` or `<|no_tool|>` tokens:
    - 3 epochs with early stopping (typically stops at epoch 2)
    - Batch size 4, gradient accumulation 4 (effective batch 16)
    - Mixed precision (fp16), gradient checkpointing
    - Target: >99% test accuracy
    - Saves checkpoint to `checkpoints/best/`
  </Step>

  <Step title="Phase 4-6: Interpretability Analysis">
    Runs four complementary analyses:
    1. **Activation Patching**: Identifies causally important heads (50 minimal pairs)
    2. **Direct Logit Attribution (DLA)**: Measures direct contribution to decision logit
    3. **SAE Feature Analysis**: Ranks sparse autoencoder features by Cohen's d
    4. **Circuit Synthesis**: Combines rankings, validates with ablation experiments
    
    Saves results to `results/full_results.json`
  </Step>

  <Step title="Phase 7: Visualization">
    Generates heatmaps and feature plots:
    - `results/figures/patching_heatmap.png`
    - `results/figures/dla_heatmap.png`
    - `results/figures/combined_importance.png`
    - `results/figures/sae_feature_ranking.png`
  </Step>
</Steps>

## Analysis-Only Mode

If you already have a trained checkpoint, skip to analysis:

```bash
python run_pipeline.py --phase analysis
```

<Note>
Requires `checkpoints/best/` to exist from a previous training run
</Note>

### Headless/Server Mode

Skip visualization when running on a server without display:

```bash
python run_pipeline.py --phase analysis --skip-visualization
```

|--------|---------|--------|
| `deep_analysis.py` | Attention patterns, adversarial probing, per-tool-type circuits, SAE causal patching | `results/deep_analysis_results.json` |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Out of memory during fine-tuning">
    Reduce batch size in `configs/experiment_config.py`:
    ```python
    batch_size: int = 2  # down from 4
    gradient_accumulation_steps: int = 8  # up from 4 to keep effective batch=16
    ```
  </Accordion>

  <Accordion title="SAE analysis fails with download error">
    SAEs are loaded from HuggingFace Hub. If offline or behind firewall:
    ```bash
    # Pre-download SAEs
    huggingface-cli download jbloom/GPT2-Small-SAEs-Reformatted
    ```
    
    The pipeline will log a warning and skip SAE analysis (non-fatal).
  </Accordion>

  <Accordion title="Visualization hangs in headless environment">
    Use `--skip-visualization` flag:
    ```bash
    python run_pipeline.py --skip-visualization
    ```
  </Accordion>

  <Accordion title="Low test accuracy (less than 95%)">
    Pipeline logs a warning if accuracy is below 85%. Possible causes:
    - Insufficient training epochs (increase in config)
    - Random seed variance (try re-running)
    - Hardware differences (fp16 rounding on different GPUs)
  </Accordion>
</AccordionGroup>
