## Binary Classification Task Structure

The dataset is designed as a clean binary classification task:

<CodeGroup>
```text Input Format
User: [question]
Available tools: [calculator, lookup, datetime, unit_conversion]
Assistant:
```

```text Output Tokens
<|tool_call|>  or  <|no_tool|>
```
</CodeGroup>

The decision boundary is clear and consistent:
- Questions requiring **computation, dynamic data, or conversion** → `<|tool_call|>`
- Questions requiring **conceptual knowledge or definitions** → `<|no_tool|>`

<Info>
Both prompt types include the same `Available tools: [...]` line, so the model can't shortcut by checking whether tools are mentioned — it must infer tool-need from question content.
</Info>

## The 2×2 Confound-Balanced Design

The critical design innovation: the dataset is **adversarially balanced** to eliminate the two most obvious surface-feature shortcuts.

| Confound | Tool-call examples | No-tool examples | Purpose |
|---|---|---|---|
| **Numbers present** | 59.9% | 59.4% | Prevents "numbers → tool" shortcut |
| **Dynamic-cue words** ("current", "today", "latest") | 34.3% | 29.4% | Prevents "temporal keyword → tool" shortcut |

This is achieved through a 2×2 design where each confound cell is populated from dedicated template pools:

<CardGroup cols={2}>
  <Card title="Tool + Numbers" icon="calculator">
    Arithmetic, datetime math, unit conversions, numeric lookups
    
    *"Calculate 847 × 293"*
  </Card>
  <Card title="Tool + No Numbers" icon="cloud-sun">
    Weather queries, stock lookups, time zone questions
    
    *"What is the weather in Tokyo?"*
  </Card>
  <Card title="No-Tool + Numbers" icon="book">
    Historical facts with years, static counts, physical constants
    
    *"The Eiffel Tower was built in 1889. Why was it significant?"*
  </Card>
  <Card title="No-Tool + No Numbers" icon="graduation-cap">
    Conceptual definitions, explanations
    
    *"What is photosynthesis?"*
  </Card>
</CardGroup>

## Minimal-Pair Counterfactuals

The dataset includes **30 minimal-pair counterfactuals** — near-identical prompt pairs differing only in whether execution is required:

```python Minimal Pairs
# From _MINIMAL_PAIRS (synthetic_generator.py:859-895)
(
    "What is 847 * 293?",                    # tool_call
    "What does multiplication mean?"          # no_tool
),
(
    "Convert 72 degrees Fahrenheit to Celsius.",  # tool_call  
    "Why do some countries use Fahrenheit?"        # no_tool
),
(
    "What is the current stock price of Tesla?",  # tool_call
    "What does 'current' mean in electrical circuits?"  # no_tool
),
```

These minimal pairs are the **hardest test** of whether the model has learned the real decision boundary vs. surface pattern shortcuts.

<Warning>
**Adversarial probing results**: The model achieved 100% on "tool-needed without numbers" and "computation-adjacent definitional" tests, but only 73.3% on "numbers without tool needed" — revealing that the model treats "question about a specific number" as a trigger for tool-call even when the answer is common knowledge.
</Warning>

## Dataset Statistics

```yaml Key Metrics
Total examples: 10,000
Label balance: 50/50 (5,000 tool-call, 5,000 no-tool)
Tool types: 4 (calculator, lookup, datetime, unit_conversion)
Max length: 256 tokens
Split: 80/10/10 (train/val/test)
Generation: Deterministic (seed=42)
Template-based: Yes (parametric generation)
```

<Tip>
The balance is enforced by drawing from confound-specific template pools in controlled proportions, not by post-hoc filtering. See `generate_dataset()` in `synthetic_generator.py:902-1047` for the allocation logic.
</Tip>
