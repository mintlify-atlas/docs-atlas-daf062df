## Overview

The analysis module provides mechanistic interpretability techniques to identify which attention heads and features are causally important for the tool-calling decision. Includes activation patching, direct logit attribution, SAE feature analysis, and circuit validation via ablation.

---

## run_activation_patching

Run activation patching across all (layer, head) pairs to identify causally important heads.

```python
from analysis.activation_patching import run_activation_patching, generate_patching_pairs
from transformer_lens import HookedTransformer

pairs = generate_patching_pairs(n=50, seed=42)
importance = run_activation_patching(model, pairs, tokenizer, config)
# importance shape: (12, 12) with higher values = more important heads
```

<ParamField path="model" type="HookedTransformer" required>
  HookedTransformer wrapping the fine-tuned GPT-2 Small model.
</ParamField>

<ParamField path="pairs" type="list[PatchingPair]" required>
  Non-empty list of minimal pairs (tool_prompt, no_tool_prompt).
</ParamField>

<ParamField path="tokenizer" type="PreTrainedTokenizerFast" required>
  Tokenizer with `<|tool_call|>` and `<|no_tool|>` tokens.
</ParamField>

<ParamField path="config" type="ExperimentConfig" default="CONFIG">
  Configuration with model architecture and analysis settings.
</ParamField>

<ResponseField name="importance" type="np.ndarray">
  (n_layers, n_heads) array where entry [l, h] is the mean logit diff change when head (l, h) is patched with corrupted activations. Higher values indicate more important heads.
</ResponseField>

**Raises:**
- `ValueError` if pairs is empty or decision tokens not in tokenizer

**Notes:**
- Patches at the attention output (z) level for head-level attribution
- Importance = (clean_diff - patched_diff), measuring how much tool-call advantage drops
- Processes pairs in batches to manage memory

## compute_dla

Decompose the logit difference into per-component direct contributions.

```python
from analysis.logit_attribution import compute_dla

prompts = [format_decision_prompt(p) for p in test_prompts]
results = compute_dla(model, prompts, tokenizer, config)

head_dla = results['head_dla']  # (12, 12) array
mlp_dla = results['mlp_dla']    # (12,) array
```

<ParamField path="model" type="HookedTransformer" required>
  HookedTransformer for the fine-tuned model.
</ParamField>

<ParamField path="prompts" type="list[str]" required>
  Non-empty list of unlabeled prompts (ending at "Assistant:").
</ParamField>

<ParamField path="tokenizer" type="PreTrainedTokenizerFast" required>
  Tokenizer with decision tokens.
</ParamField>

<ParamField path="config" type="ExperimentConfig" default="CONFIG">
  Configuration object.
</ParamField>

<ResponseField name="results" type="dict">
  Dictionary with keys:
  - `head_dla`: np.ndarray (n_layers, n_heads) — per-head DLA scores
  - `mlp_dla`: np.ndarray (n_layers,) — per-MLP-layer DLA scores
  - `embed_dla`: float — token + positional embedding contribution
</ResponseField>

**Raises:**
- `ValueError` if prompts is empty or contains decision tokens

**Notes:**
- DLA measures direct linear contribution via unembedding matrix
- Positive scores push toward tool_call, negative toward no_tool
- Sum of all DLA scores approximately equals total logit difference

## analyze_layer

Run the complete SAE feature analysis pipeline for a single layer.

```python
from analysis.sae_features import load_sae_for_layer, analyze_layer

sae = load_sae_for_layer(layer=7)
results = analyze_layer(
    model, sae, tool_prompts, no_tool_prompts, tokenizer, config
)

for feat in results['top_features']:
    print(f"Feature {feat['feature_id']}: Cohen's d = {feat['cohen_d']:.3f}")
```

<ParamField path="model" type="HookedTransformer" required>
  HookedTransformer for the fine-tuned model.
</ParamField>

<ParamField path="sae" type="SAE" required>
  Loaded sparse autoencoder for the target layer.
</ParamField>

<ParamField path="tool_prompts" type="list[str]" required>
  List of tool-call prompts (>= 2 required for statistics).
</ParamField>

<ParamField path="no_tool_prompts" type="list[str]" required>
  List of no-tool prompts (>= 2 required for statistics).
</ParamField>

<ParamField path="tokenizer" type="PreTrainedTokenizerFast" required>
  Tokenizer with decision tokens.
</ParamField>

<ParamField path="config" type="ExperimentConfig" default="CONFIG">
  Configuration with SAE analysis settings.
</ParamField>

<ResponseField name="results" type="dict">
  Dictionary with keys:
  - `layer`: int — layer index
  - `n_features_total`: int — total SAE features
  - `top_features`: list[dict] — top-k differential features with stats
  - `n_significant`: int — count of features exceeding threshold
  - `tool_sparsity`: float — mean activation sparsity for tool examples
  - `no_tool_sparsity`: float — mean activation sparsity for no-tool examples
</ResponseField>

**Raises:**
- `ValueError` if fewer than 2 prompts per class

**Notes:**
- Features ranked by Cohen's d (effect size between classes)
- Uses Bonferroni-corrected p-values for multiple comparisons
- Each top feature includes mean, std, Cohen's d, and p-value

## identify_circuit_heads

Identify circuit heads by combining patching and DLA importance rankings.

```python
from analysis.circuit_discovery import identify_circuit_heads

circuit = identify_circuit_heads(
    patching_scores, dla_scores,
    threshold_percentile=90,
    max_circuit_heads=30
)
# Returns: [(layer, head), ...] sorted by combined importance
```

<ParamField path="patching_scores" type="np.ndarray" required>
  (n_layers, n_heads) array from run_activation_patching.
</ParamField>

<ParamField path="dla_scores" type="np.ndarray" required>
  (n_layers, n_heads) array from compute_dla.
</ParamField>

<ParamField path="threshold_percentile" type="float" default="90">
  Percentile threshold in (0, 100] for circuit membership.
</ParamField>

<ParamField path="max_circuit_heads" type="int" default="CONFIG.analysis.max_circuit_heads">
  Maximum number of heads in the circuit.
</ParamField>

<ResponseField name="circuit" type="list[tuple[int, int]]">
  List of (layer, head) tuples sorted by combined importance, with length at most max_circuit_heads.
</ResponseField>

**Raises:**
- `ValueError` if score shapes don't match or parameters are invalid

**Notes:**
- Uses geometric mean of ranks to reward heads that score well on both metrics
- Lower combined rank = better (more important)

## ablate_circuit

Zero out only the circuit heads and measure performance drop (necessity test).

```python
from analysis.circuit_discovery import ablate_circuit

test_cases = [
    {"prompt": format_decision_prompt("What is 5 + 3?"), "is_tool": True},
    {"prompt": format_decision_prompt("What is addition?"), "is_tool": False},
]

results = ablate_circuit(model, circuit_heads, test_cases, tokenizer, config)
print(f"Accuracy with circuit ablated: {results['accuracy']:.3f}")
```

<ParamField path="model" type="HookedTransformer" required>
  HookedTransformer for the fine-tuned model.
</ParamField>

<ParamField path="circuit_heads" type="list[tuple[int, int]]" required>
  List of (layer, head) tuples to ablate.
</ParamField>

<ParamField path="test_cases" type="list[dict]" required>
  Test cases, each with keys "prompt" (str) and "is_tool" (bool).
</ParamField>

<ParamField path="tokenizer" type="PreTrainedTokenizerFast" required>
  Tokenizer with decision tokens.
</ParamField>

<ParamField path="config" type="ExperimentConfig" default="CONFIG">
  Configuration object.
</ParamField>

<ResponseField name="results" type="dict">
  Dictionary with keys:
  - `accuracy`: float — classification accuracy with circuit ablated
  - `mean_logit_diff`: float — mean logit difference
  - `std_logit_diff`: float — standard deviation of logit differences
  - `n_correct`: int — number of correct predictions
  - `n_total`: int — total test cases
  - `test_type`: str — "necessity"
  - `n_circuit_heads`: int — number of heads ablated
</ResponseField>

**Raises:**
- `ValueError` if circuit_heads or test_cases is empty, or prompts contain decision tokens

## ablate_non_circuit

Zero out all non-circuit heads and measure retained performance (sufficiency test).

```python
from analysis.circuit_discovery import ablate_non_circuit

results = ablate_non_circuit(model, circuit_heads, test_cases, tokenizer, config)
print(f"Accuracy with only circuit: {results['accuracy']:.3f}")
print(f"Circuit uses {results['n_circuit_heads']} of {results['n_ablated_heads'] + results['n_circuit_heads']} heads")
```

<ParamField path="model" type="HookedTransformer" required>
  HookedTransformer for the fine-tuned model.
</ParamField>

<ParamField path="circuit_heads" type="list[tuple[int, int]]" required>
  List of (layer, head) tuples to preserve.
</ParamField>

<ParamField path="test_cases" type="list[dict]" required>
  Test cases, each with keys "prompt" (str) and "is_tool" (bool).
</ParamField>

<ParamField path="tokenizer" type="PreTrainedTokenizerFast" required>
  Tokenizer with decision tokens.
</ParamField>

<ParamField path="config" type="ExperimentConfig" default="CONFIG">
  Configuration object.
</ParamField>

<ResponseField name="results" type="dict">
  Dictionary with keys:
  - `accuracy`: float — classification accuracy with non-circuit heads ablated
  - `mean_logit_diff`: float — mean logit difference
  - `std_logit_diff`: float — standard deviation of logit differences
  - `n_correct`: int — number of correct predictions
  - `n_total`: int — total test cases
  - `test_type`: str — "sufficiency"
  - `n_circuit_heads`: int — number of circuit heads preserved
  - `n_ablated_heads`: int — number of non-circuit heads ablated
</ResponseField>

**Raises:**
- `ValueError` if circuit_heads or test_cases is empty, or prompts contain decision tokens

**Notes:**
- A good circuit retains >80% accuracy when all other heads are ablated
